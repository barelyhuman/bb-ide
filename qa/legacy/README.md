# Legacy QA

This folder holds older umbrella runbooks while their scenarios are being mapped into the new surface-based QA structure.

Use `legacy/` only when:

- the old doc still contains valuable coverage that has not been fully remapped
- ownership between surfaces is still ambiguous

When a scenario is clearly covered by a new surface doc, delete the legacy copy instead of keeping two sources of truth.
